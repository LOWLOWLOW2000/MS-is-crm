# アカウント作成とストレージ接続

ログイン画面（`/login`）では、**アカウント作成時にオンラインストレージまたはローカルストレージの接続を必須**としています。

## どのようなアカウント・設定にすればよいか

- **API認証が用意されているストレージ**でアカウント用データを保持することを推奨します。
- 認可・アクセス制御をAPIで扱えるため、セキュリティと運用を抑えやすくなります。

### 推奨ストレージ例

| 種別 | 例 | API認証 |
|------|-----|--------|
| オンライン | Google ドライブ | Google Drive API（OAuth 2.0） |
| オンライン | Microsoft OneDrive | Microsoft Graph API（OAuth 2.0） |
| オンライン | Dropbox / Box など | 各社 REST API + OAuth |
| ローカル | 自前ストレージ + REST API | API キー / JWT など |

- **Google ドライブ**: クラウドコンソールでプロジェクト作成 → Drive API 有効化 → OAuth 2.0 クライアント ID を取得。
- **Microsoft OneDrive**: Azure AD でアプリ登録 → Microsoft Graph（Files スコープ）で認可。
- **その他**: ストレージの読み書きを REST API で提供し、API キーや JWT で認証する構成。

### 実装時の方針

- ログイン/アカウント作成フローで「ストレージ接続」ステップを必ず挟む。
- 接続先は Google / OneDrive / その他（API認証あり）のいずれかで選択可能にする。
- トークンや接続情報はバックエンドで安全に保持し、フロントでは必要最小限のスコープのみ扱う。

現状のログイン画面では、上記を案内しつつストレージ選択ボタンは「準備中」として表示しています。実装時は各プロバイダの OAuth または API 認証フローを組み込んでください。
