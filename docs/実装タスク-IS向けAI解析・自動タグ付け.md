# 実装タスク: インサイドセールス向けAI解析・自動タグ付けシステム

## 1. 概要
Zoom Phoneの録音・文字起こし完了Webhookをトリガーに、Gemini 1.5 Flashを使用して「個人情報のマスク化」「商談ランク付け」「100個のタグ判定」を一括で行い、**テナント側ストレージにある通話ログを一時的に解析し、その結果（マスク済みテキスト＋抽象化された分析JSON）のみを運営側DBに保存する**機能を実装する。

## 2. システムアーキテクチャ
1. **Webhookエンドポイント**: Zoom Phoneからの `phone.recording_completed` を受信。
2. **データ取得**: `download_url` を使用して文字起こし(transcript)データを取得。
3. **AI解析 (Gemini 1.5 Flash)**:
   - 1リクエストで「マスク処理」と「分析」を同時に実行。
   - 有料版(Pay-as-you-go)を利用し、データ学習を回避。
4. **データ保存**:
   - 音声ファイル・元の文字起こし: 原則として各テナントが管理するストレージ（Google Drive / 顧客S3 / 顧客Supabase 等）に保存。
   - 運営側DB: マスク済みテキストと分析結果(JSON)のみを暗号化して保存（設計上は「分析結果(JSON)のみ」でも成立する構造とする）。

## 3. 実装用プロンプト (System Instruction)
以下のプロンプトをGemini APIのシステム指示として組み込む。

```text
あなたは高度なセールスアナリストです。
以下の手順で入力されたセールストークの文字起こしを処理し、JSON形式で返してください。

【1. 匿名化処理 (Anonymization)】
- 個人名、個人の電話番号、詳細な住所を特定し、`[PERSON]`, `[PHONE]`, `[ADDRESS]` に置換すること。
- ただし、分析に必要な「企業名」「業種」「部署名」「役職」「興味を示した商材名」は保持すること。

【2. 商談分析 (Analysis)】
- 商談ランク判定: A（即案件化）, B（時期指定あり）, C（情報収集）, D（対象外）
- 関心商材カテゴリの特定: （例：セキュリティ、人事SaaS、インフラ等）
- 100個のタグ判定: 事前に定義されたリストに基づき、該当するものを抽出。

【3. 出力形式 (JSON)】
{
  "masked_transcript": "（マスク済み全文）",
  "analysis": {
    "lead_score": "A",
    "target_department": "IT推進部",
    "interest_category": "セキュリティ系SaaS",
    "tags": ["予算確認済", "競合A社利用中", "決裁権あり"],
    "summary": "（運営・開発向けの3行要約）"
  }
}
```

## 4. 本プロジェクトとの対応

- **Zoom連携**: 現行 `.cursorrules` では ZOOM Meeting + クラウド録音 → 1日2回バッチDL → Whisper + GPT-4o-mini を想定。本タスクは Zoom **Phone** + Webhook即時処理 + **Gemini 1.5 Flash** の別フローとして、構想に併記する。
- **100タグ**: `.cursorrules` の「AI音声評価タグ設計（100種類）」とタグリストを揃えるか、本システム用のサブセットを定義する。
- **DB**: マスク済みテキスト・分析結果の保存は、既存の `calling-ai-evaluation` 等のスキーマ拡張または新エンティティで対応する。いずれの場合も、**個人名・個人直通・詳細住所などのPIIはAI側でマスク済み**であり、運営側DBには保存しない。

## 5. SaaS版IS-CRMとのデータ責任分離ポリシー

- SaaS版IS-CRMでは、**個人情報を含む通話ログ（音声・全文文字起こし）はテナント側ストレージに保持**することを原則とし、運営側は以下のみを保持する:
  - 公式に取得した「会社情報」（会社名・代表電話・住所など）
  - 匿名化済みの「商談タグ」「評価スコア」「要約」などの抽象化データ
- `apps/ai` からは、テナント側ストレージへの署名付きURLやアクセストークンを用いて一時的に文字起こしを取得し、処理完了後は原文を保持しない。
- AIクロール（評価バッチ）の**実行タイミング・対象ボリューム・利用モデル（GPT-4o-mini / Gemini 等）**は、将来的に運営UIから設定可能とし、推定トークン数と想定ドル／円コストを事前に表示できるようにする。***
