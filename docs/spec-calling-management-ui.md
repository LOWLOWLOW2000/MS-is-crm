# 架電管理UI 仕様書

**v2 の見方**  
架電画面（`/calling`）を開き、**右上の「UI: 1 | 2」** で **2** を押すと本仕様の v2 UI（モック）が表示されます。**1** で現行 UI に戻ります。

URL（現行）: http://localhost:3000/calling  
※ポート 3002 で提供する場合は、Web の起動オプションで `-p 3002` を指定してください。

---

## ヘッダーセクション

### タブ切替
- **個人 / チーム / 全体**
- **当日 / 週 / 月**

### KPIカード（グラフィカル・大型表示）
※オフィス掲示用グラフィカルデザイン。インサイドセールスが最優先で確認する数字。  
※現在値：大きく表示　／　目標値・達成率：小さく表示  
※既存アセットにないグラフィカル演出は実装しない

| 指標     | 現在値 | 目標  | 達成率 |
|----------|--------|-------|--------|
| 架電数   | 0      | 15    | 0%     |
| アポ     | 3      | 5     | 60%    |
| KEY接続  | 8      | 10    | 80%    |
| 資料送付 | 2      | 3     | 67%    |

---

## 左カラム

### 上部セクション：リスト一覧（看板パネル）

- アイコン＋リスト名 の看板スタイルで並べる
- クリックすると価格.com・REINZスタイルの2列リスト表示に展開
- 各行の左端に縦並びのアクションボタン：
  - **修正ボタン**
  - **リスト表示ボタン**（該当情報をメインUIに表示し、ポップアップを閉じる）

---

### リスト詳細パネル（看板パネル式・薄枠・角なし）
※会社のレイヤー構造が雰囲気で把握できるよう、パネルの入れ子構造で表現

**表示項目（記載がある場合のみ表示）：**
- 会社名（法人名）

**電話一覧パネル（横並びパネル式）：**
- 上段：部署名
- 下段：電話番号
- 選択すると → 下記「リスト内容確認フロー」のセット先にセットされる

---

### リスト内容確認フロー（同一DIV内）

**ステート①：リスト内容未確認**（初期状態）
- チェック完了後にボタンを押す
- ボタン名：「リスト☑完了」

**ステート②：リスト内容承認済み**（確認完了後）
- 架電機能に切り替わる
- 表示要素：
  - 電話発信・着信ボタン（メイン）
  - ミュートボタン（横に配置）

**フッター操作：**
- 一番左：「← リストに戻る」ボタン

## 現行実装との主な差分（参照用）

| 項目 | 本仕様書 | 現行 is-crm `/calling` |
|------|----------|-------------------------|
| URL/ポート | localhost:3002/calling | localhost:3000/calling |
| ヘッダー | タブ（個人/チーム/全体、当日/週/月）＋KPIカード4種 | なし（リスト選択＋ステータスのみ） |
| KPI | 架電数・アポ・KEY接続・資料送付（目標・達成率付き） | レポート別画面で集計、架電画面にはなし |
| リストUI | 看板パネル→2列リスト展開、修正・リスト表示ボタン | ドロップダウンでリスト選択、リスト内は架電画面右で1件ずつ |
| リスト詳細 | 会社名＋電話一覧（部署/番号）、選択で発信先にセット | 左ペインに会社名・電話・住所・承認→発信 |
| 確認フロー | 「リスト☑完了」→ 発信/着信/ミュート | 「内容確認・承認」→ ZOOM発信（人間承認スイッチ） |
| 電話データ構造 | 部署＋電話番号の複数（横並びパネル） | 1件あたり1電話（ListItem.phone） |

※上記は実装のずれを整理するためのメモ。本仕様に合わせる場合は API・型・画面構成の変更が必要。

## v2 用 API メモ（モック後の本実装用）

v2 UI は当面 **モックデータのみ** で動作。以下の API は将来、SFA またはアキナイ側で用意する際の検討用。

### ヘッダー・KPI

| 用途 | 想定API | メモ |
|------|---------|------|
| 範囲タブ（個人/チーム/全体）× 期間タブ（当日/週/月）の KPI 取得 | `GET /calling/kpi?scope=personal\|team\|all&period=today\|week\|month` | 架電数・アポ・KEY接続・資料送付の現在値・目標値・達成率。目標はテナント/ロールで設定可能に。 |
| 目標値の取得・更新 | `GET /settings/kpi-goals` / `PATCH /settings/kpi-goals` | developer または is_admin が編集。 |

### 左カラム・リスト

| 用途 | 想定API | メモ |
|------|---------|------|
| リスト一覧（看板用） | 既存 `GET /lists/assigned/me` または `GET /lists` を拡張 | アイコン＋リスト名。v2 用に「看板スタイル」で返すか、フロントで整形。 |
| リスト内アイテム（2列リスト用） | 既存 `GET /lists/:listId/items` を拡張 | 1件あたり **会社名 + 電話一覧（部署・番号の配列）** が必要。現行は `ListItem.phone` 1本。部署・複数番号対応はスキーマ拡張または別エンドポイント。 |
| 修正 | `PATCH /lists/:listId/items/:itemId` または同等 | 会社名・電話一覧の更新。 |
| リスト表示 | 実体は「選択したリスト・アイテム・電話をメイン表示にセット」するだけなので、API 不要（状態のみ）。 |

### リスト内容確認フロー

| 用途 | 想定API | メモ |
|------|---------|------|
| リスト☑完了 | 現行の「内容確認・承認」に相当する場合は `POST /calling/approvals` を流用可能。 | v2 は「リスト内容を確認した」という意味なので、必要なら `POST /calling/list-confirm` のような軽い記録だけでも可。 |
| 発信・着信・ミュート | 発信は現行 `POST /calling/dial-check` → ZOOM 等。着信・ミュートは端末/WebRTC 依存。 | 既存架電 API と統合するか、v2 専用の薄いラッパーを検討。 |

### データ型（v2 向け拡張例）

- **ListItem 拡張**  
  - `phone: string` のまま「代表」1本だけ持つ + **`phones: { department: string; number: string }[]`** を追加する形が扱いやすい。  
  - または `ListPhone` を別テーブルにして 1 対多で紐付け。

- **KPI 集計**  
  - 架電数: `CallingRecord` の件数。  
  - アポ: 結果タグが「アポ」系の件数。  
  - KEY接続・資料送付: 結果タグまたは別フラグで定義し、集計 API で返す。

現時点では **モックで UI のみ実装**。上記は「ボタンで中身が変わるギミック」を本 API に差し替える際の参照用。
