## 現在の状況と予定（2025年3月〜）

### 何が変わったか

| 要因 | 内容 |
|------|------|
| **UI の完全リニューアル** | 00-INDEX の通り、一度画面・ナビ・ダッシュボード等を削除し、**ui-structure.md**（MGIS01 スプレッドシート準拠）を正として作り直す方針。 |
| **ルール・仕様の見直し** | .cursorrules やプロジェクトルールの使用変更。新 UI は「トップ → ログイン → ダッシュボード（左ナビ＋ヘッダ＋メイン＋フッター）」を骨格とし、営業ルーム・ディレクター・運営バックヤードの URL を ui-structure に合わせて再定義。 |
| **予定感の変化** | 上記のため、**Phase 1 で「完了」としていた Web 側の画面**（架電専用UI・リスト管理・レポート・運営ページなど）は、**旧 UI ツリー上**の実装。新 UI モックは**ルート・レイアウト・ナビ・各ページの「枠」**までを整備した段階で、**中身の実装・API との接続はこれから**。 |

### 現在のモック（新 UI）の状態

- **できていること**
  - トップ（/）・ログイン（/login）・ダッシュボード共通レイアウト（ヘッダ・左ナビ・フッター・メイン）。
  - ダッシュボードハブ（/dashboard）からの全ルートへのリンクと、左ナビ（DashboardNav）での一覧。
  - 営業ルーム・未調査リスト精査・運営（/ops）の**骨格・レイアウト**。
  - ui-structure.md に記載された URL のほぼすべてに対応する page の存在（多くは見出し＋説明のプレースホルダ）。
- **まだのもの**
  - 各画面の中身（KPI・AI日報・ディレクター系・企業管理者・ops 各サブなど）の実装。
  - 認証・リスト・架電・レポートなど**既存 API と新 UI の接続**。
  - 旧 Phase 1 残タスク（右ペインリサイズ・ユーザー一覧ページ・レポート IS 別表示）は、**新 UI に組み込む形で再度実施**する想定。

### 今後の進め方（予定の整理）

1. **新 UI を正とする**: 今後の機能追加・残タスクは **ui-structure.md の画面構成** を前提に進める。
2. **Phase 1 の「API と旧 Web」**: 認証・リスト・架電・ZOOM・レポート等の **API と旧架電画面** はそのまま参照可能。新 UI 側からこれらを呼ぶ形で接続する。
3. **残タスクの再マッピング**:
   - 架電のリサイズ・ユーザー一覧・レポート IS 別表示 → **新 UI の該当ページ**（架電は 営業ルーム 内、管理は ops/members 等）に実装する。
   - KPI・AI日報・日報BOX・ディレクター系・運営サブなど → **新 UI のプレースホルダを順次実装**。

上記により、**「Phase 1 完了状況」は過去のスナップショットとして残しつつ、現在の予定は新 UI とルール変更に合わせてこの節で更新する**形にします。
