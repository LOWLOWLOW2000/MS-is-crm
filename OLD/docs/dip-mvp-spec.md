# インサイドセールス共通 架電仕様（1人MVP）

コールドコール主体のテレアポに共通する最小仕様。IS_01 は dip・不動産・ノマドIS など案件を問わず同じUIで運用し、展開しやすくする。

---

## 設計の考え方（素人向け）

**「DIPモードは廃止。常に汎用11種だけを画面で選び、そのまま保存する。先方（dip等）に渡すときだけエクスポートでマッピング。」**

### イメージ

- **画面もデータも「汎用11種」だけ**  
  担当者あり興味・担当者あり不要・不在・番号違い・断り・折り返し依頼・留守電・資料送付・アポ・リスト除外・不通。モード切替はない。
- **先方ごとの名前（例：dipの「トスアップ」「キーマンNG」）は、ツールには出さない。**  
  CSVやCRMに渡すときだけ「汎用 → 先方の表示名」に変換する（エクスポート／マージ時のマッピング）。

### なぜこうするか

- **MVP達成と展開** … モードを分けるとPMVが分断され、他案件・他CRMへの展開が遅れる。1本のフローにしておく。
- **正本は1種類** … 汎用だけ保存すれば、dip・不動産・その他どこにも同じデータを、その先の言い方に変えて渡せる。

## 4. 実装方針（IS_01 での対応）

- **データは汎用で保持**: 架電記録・アポ（日時・取得時の作業ステータス）・メモを「共通の形」で保存する。DIPの11種は**保存する項目としては持たない**（上記「設計の考え方」参照）。
- **入力UI**: 画面は常に**汎用11種**のボタンのみ。選択した値がそのまま保存される（モード切替なし）。
- **エクスポート／先方CRMマージ**: 保存済みの汎用データを、先方（dip・不動産等）が求める列・表示名に**マッピング**してCSVやAPIで渡す。
- **結果タイプ**: `CallingRecord.result` には上記§5の汎用11種のみを保存する。DIPの11種は保存しない。
- **リスト**: 既存 `CallingList` / `ListItem` を利用。リストNoは `ListItem.id`。カテゴリ・エリア・業種・役職・担当者名・架電回数・架電履歴・テンプレ（アポ時）は、必要に応じて `ListItem` の拡張またはメタJSONで保持。
- **アポ時テンプレ**: 既存のスクリプト/テンプレ（管理者がセット）を「アポ」選択時に表示し、入力内容をメモまたは専用フィールドに保存。
- **CSV出力**: 架電結果一覧を、dip指定列（リストNo・企業名・電話・結果・メモ・アポ時入力・架電日時など）でエクスポート。

## 6. 先方別エクスポートマッピング（例：dip）

※画面では汎用11種のみ。**先方に渡すときだけ**、汎用 → 先方の表示名に変換する。

### 例：汎用結果 → dipが求める表示ラベル

| 保存されている汎用結果 | dip用CSV等での表示例   |
|------------------------|------------------------|
| 担当者あり興味         | トスアップ／ヒアリングできた 等 |
| 担当者あり不要         | キーマンNG／受付NG 等  |
| 資料送付               | 資料送付(受付)         |
| 不在                   | キーマン不在          |
| 不通                   | 不通                  |
| 断り                   | 現アナ 等             |
| リスト除外             | リスト除外            |
| アポ                   | アポ                  |
| 折り返し依頼           | ヒアリングできず 等   |

- **エクスポート時**: 保存済みの汎用結果を、上表のような先方ごとのルールでCSV・APIに変換する。

---

## 7. 架電一覧（時間帯別）

- **一覧**: 架電対象をリストで表示する（「今日かける1件」ではなく一覧）。
- **時間帯別**: 午前／午後など時間帯で絞り込みまたはグループ表示する。例：9–12時・12–14時・14–18時など、案件ごとに設定可能なスロットで一覧を切り替えまたはセクション分けする。
- 一覧の列：リストNo・企業名・電話番号・カテゴリ・エリア・業種・架電回数・架電履歴・希望時間帯（任意）など。行クリックで企業詳細＋行動結果パネルへ。

---

## 8. 導線（1人MVP）

- **営業ルーム**: `/sales-room` で常に同じUI。先頭タブが「一覧」、行動結果は汎用11種のみ。モード切替なし。
- **架電一覧（時間帯別）**: 一覧タブで時間帯を切り替え／グループ表示。リスト連携後に実データを表示。
- **CSV出力**: 架電記録をAPIで保存した上で、架電結果一覧から先方（dip等）用にマッピングしてエクスポートする機能を次フェーズで実装。
