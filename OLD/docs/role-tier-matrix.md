## マトリクス

| カテゴリ / 内部リンク | TIER 1 (企業管理者) | TIER 2 (ディレクター) | TIER 3 (ISリーダー) | TIER 3 (IS) | 備考 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **権限委任** | ◯ | ◯ | - | - | 下位Tierへの仕組み委任権 |

**権限強度（パワーバランス）:**

```
左 (TIER 1) <------------------------------------------------> 右 (TIER 3)
      強                                                       弱
```

## アプリロールとの対応

| アプリの role (JWT) | マトリクス上の扱い |
|--------------------|---------------------|
| `developer` | TIER 1 相当（全リンク表示） |
| `enterprise_admin` | TIER 1（企業管理者） |
| `is_admin` | TIER 1（企業管理者） |
| `director` | TIER 2（ディレクター） |
| `is_member` | TIER 3（IS） |

- **TIER 3 (ISリーダー)**  
  - マトリクス上は「営業ルーム・個別KPI・日報・アカウント」が ◯ で、権限委任・プロジェクトKPI・日報（受け）・リスト配布は ×。  
  - 現状の実装では **`is_member` と同じ表示**（TIER 1・2 専用項目は非表示）。  
  - 将来、ISリーダー専用の機能（例：チーム内の簡易集計）を出す場合は、`UserRole` に `is_leader` を追加し、ここに `is_leader` → TIER 3 (ISリーダー) の行を追加する。

## 表示ルール（左ナビ・メニュー）

- **TIER 1・2 のみ表示**: 権限委任、プロジェクトKPI、AIレポート、日報BOX、リスト配布  
  → `enterprise_admin` / `is_admin` / `director`（＋開発時は `developer` でメニューから全表示）
- **全員表示**: メニュー、カレンダー、アカウントページ、個別KPI、AI日報、営業ルーム  
  → `is_member`（および将来の `is_leader` も同様）
- **企業管理者のみ表示（ページ内外の全リンク）**: ディレクター、運営、メンバー・ロール管理、未調査リスト精査、社内報（外部・予定）、ログアウト  
  → `enterprise_admin` / `is_admin` のとき左カラムに上記を追加表示
