# SFAフロー 現状レビュー（手入力の摩擦ポイント）

## 対象

- バックエンド: `apps/api/src/calling`, `apps/api/src/lists`
- フロント: `apps/web/app/(dashboard)/sales-room`（セールスルーム・電話システム）, `apps/web/app/(dashboard)/recall`（再架電）

## 摩擦ポイント（3つ）

### 1. 保存と「次へ」が別ボタンで2アクション必要

- 現状: 「上書き保存」押下後に「次へ」を押す必要があり、1件あたりクリックが2回になる。
- 影響: リスト消化が速い現場では操作回数が多く、保存し忘れて次へ進むリスクもある。

### 2. 次回架電日時が毎回ゼロから入力

- 現状: `nextCallAt` は datetime-local で空のまま。結果種別に応じたデフォルトがない。
- 影響: 「不在」「折り返し依頼」「留守電」など結果ごとに典型的な間隔があるが、毎回手入力になる。

### 3. 結果選択がラジオのみでキーボードショートカットなし

- 現状: 結果は7種類のラジオボタンのみ。マウスで選択する前提。
- 影響: 手が離せない／リストを連続で捌く際に、キーボードで結果→保存→次へと流せない。

---

## 補足（バックエンド・導線）

- `POST /calling/records` と `GET /calling/recall` は tenantId フィルタ・ソートともに実装済み。
- リスト配布・配布済み items 取得は role と tenantId で制御済み。
- 再架電一覧から「架電画面へ」で `?company=...&url=...` 遷移し、手入力用の企業情報が復元される流れは実装済み。
