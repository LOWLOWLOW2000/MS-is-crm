# 企業データベース・論理ツリー と 現行DB の対応状況

## 論理ツリー（目標モデル）

| 階層 | 名称 | 主な内容 |
|------|------|----------|
| **L1** | 企業グループ（Ultimate Parent） | グループ全体与信・系列情報、資本関係フラグ（上場・非上場） |
| **L2** | 法人エンティティ（Legal Entity / 法人番号単位） | 商号、本店所在地、設立日、資本金、売上高、営業利益、決算期 |
| **L3** | 拠点・事業所（Establishment） | 所在地、拠点種別（営業所、製造、R&D） |
| **L4** | 組織・部署（Department） | 部署名、役割分類（営業、人事、情報システム） |
| **L5** | 担当者・キーマン（Persona） | 役職ランク、決裁権限、コンタクト履歴（2-31-54 等） |
| **L2並列** | 動的インテント層 | 採用ログ、テクノロジースタック、パブリシティ、独自スコア（Sampo-Yoshi、DX成熟度、IS可機率） |

---

## 現行DBとの対応

**結論: 現状のDBは上記論理ツリーに対応していない。**

### 現行で「企業」に相当するもの

- **ListItem**（`list_items`）  
  - リスト1件 = **フラットな1行**  
  - 項目: 企業名・電話・住所・ターゲットURL・業種タグ  
  - グループ／法人／拠点／部署／担当者の階層・リレーションは**なし**
- **CallingRecord**（`calling_records`）  
  - 架電履歴。企業名・電話・住所・URL を**都度スナップショット**で保持  
  - 企業マスタ（L1〜L5）への参照は**なし**

### ギャップ一覧

| 論理ツリー | 現行DB | 備考 |
|------------|--------|------|
| L1 企業グループ | **なし** | 与信・上場/非上場等のテーブルなし |
| L2 法人エンティティ | **なし** | 法人番号・商号・本店・設立日・資本金・決算・売上・利益等なし |
| L3 拠点・事業所 | **なし** | 拠点・支店・工場のエンティティなし |
| L4 組織・部署 | **なし** | 部署・役割分類のエンティティなし |
| L5 担当者・キーマン | **なし** | 役職ランク・決裁権限・コンタクト符号のエンティティなし |
| L2並列 動的インテント | **なし** | 採用ログ・Techスタック・パブリシティ・独自スコア用テーブルなし |

現行は「リスト＝企業名＋連絡先＋URL＋業種」の**フラットなリスト**と、その**架電結果**のみを扱う構成。
