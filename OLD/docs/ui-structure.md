# UI 構成（完全リニューアル用）

MGIS01 スプレッドシートをベースにした画面・レイアウトの定義。実装はこのドキュメントを正とする。

---

## 1. 全体フロー

```
Step 1: トップ
  ↓
Step 2: ログイン  ←→  開発者ログイン（暫定・いずれ削除） → 開発ページ
  ↓
Step 3: ダッシュボード（左ナビ + ヘッダー + メイン + フッター）
```

- **トップ（Step 1）**: 商品紹介兼ねた入口・LPスタイル。コピー「人類唯一の差別化は一次情報です」の下に **MG_01入室** ボタン。背景白。文字はエヴァインスイア（フォント指定は厳密でなくてよい）。
- **ログイン（Step 2）**: OAuth認証（Google）・メール認証。開発テスト中はテキストでIS・ディレクター・メアド・パスを分かりやすく記載。
- **開発者ログイン**: 開発用。成功後は ID / PASS にて開発ダッシュボードへ。**暫定機能で、いずれ削除する想定。**
- **ダッシュボード（Step 3）**: ログイン後の共通レイアウト（左・ヘッダー・フッター・メイン）。
---

## 2. ダッシュボード レイアウト

| 領域 | 内容 |
|------|------|
| **ヘッダ** | 共通ヘッダ。個人アカウント（正式氏名・表示名・アイコン写真）・メール・パス確認変更・紐付けGoogleアカウント変更 等 |
| **左（DB 左）** | ナビゲーションリンク一覧（下記「左ナビ項目」） |
| **メイン** | 左カラムの各子ページへ |
| **フッター** | 共通フッター |

## 3. 左ナビ項目（ダッシュボード 左）

### 共通・IS向け

| 表示名 | 備考 |
|--------|------|
| schedule（ヘッダ） | 当月の総稼働時間/予定時間、日程カレンダーとタイムカード |
| 個人アカウントページ | 個人ページ・メール・パス確認変更、紐付けGoogleアカウント変更 |
| KPIページ | PJ毎のISは自分が参加するPJ。ディレクター以上は全PJを選択可能。KPIピボットテーブルで個人～チーム～プロジェクト毎に見れる仕様。もっと簡単で同じことが出来るなら提案可。 |
| AI日報 | PJを選び「生成開始」で生成。直接修正してから投函。ポストするとディレクター以上が見れる日報BOXへ。 |

### AI日報画面レイアウト（1画面に縦並び）

- **ヘッダ**: 案件選びプルダウン・過去提出の一覧ページリンク
- **メインボディ1**: その人の当日KPI・AI分析ボタン（押すとメインボディ2に結果表示）
- **メインボディ2**: 定量的報告・問題点明記・対策・自由記述欄に対してのアドバイス（修正可能な分析とアドバイス）
- **スレッドセクション**: ライン的なスレッドでディレクターたちと返事・追加質問ができる仕様

## 7. ブロックレイアウト（グリッド仕様）

全体を **10 カラム** のグリッドで組み、以下をブロックで分割する。

| ブロック | カラム幅 | 内容 |
|----------|----------|------|
| **header** | 10 | ヘッダー（内側は 3 カラム） |
| **Lber**（左バー） | 4 | リスト選択・電話制御、1企業簡易表示×4 |
| **smalleheader**（メイン） | 6 | 企業詳細・HP・トーク・案件ルール・KPI・日報・自由メモ |
| **group2**（フッター） | 10 | フッター |

### 7.1 ヘッダー（block:header）

- **columns 3** で横並び。
  1. アカウントアイコン
  2. コモンズとマリシからのお知らせ（無料枠は広告）
  3. ログアウト

### 7.2 左バー（block:Lber）

- **columns 1**（縦積み）。
- リスト選択・電話制御
- 1企業：簡易表示1～4

### 7.3 メイン（block:smalleheader / smallhead）

- 企業詳細・企業HP・トーク・案件ルール・KPI・日報・自由メモ（個人用/案件用）をタブまたは縦積みで表示。

### 7.4 フッター（block:group2）

- フッター共通（10 カラム幅）。

## 8. ページ移管図

```
Step 1: トップ (/)
  ↓
Step 2: 認証 (/login)  ※開発者ログインは暫定
  ↓
Step 3: ダッシュボード (/dashboard)
  ├→ 企業管理者 (/dashboard/corporate)
  ├→ ディレクター (/dashboard/director)
  │     ├→ プロジェクトKPI (/dashboard/director/kpi)
  │     ├→ AIレポート (/dashboard/director/ai-report)
  │     └→ 日報BOX (/dashboard/director/daily-box)
  └→ 営業ルーム（セールスルーム）(/dashboard/sales-room)

運営ダッシュボード (/ops)  ※バックヤード
  └→ 課金リミッター / 課金plan調整 (/ops/billing) 等
```

- 一般ユーザー: トップ → 認証 → ダッシュボード → ロール別ページ。
- 運営者: 運営ダッシュボード → 課金・プラン調整等。

## 9. ワイヤー詳細（実装準拠）

### 9.1 共通レイアウト（全画面）

- **ヘッダ**（全幅）
- **カラム**（左サイドバー）＋ **メイン**（右）
- **フッタ**（全幅）

### 9.2 営業ルーム（/dashboard/sales-room）

- **ヘッダ内**: 営業ルーム、アカウントアイコン・メニュー、役職変更ボタン、AI house、当日KPI（個人・チーム・PJ切り替え）
- **左カラム**: 電話制御、« ページ送り »、リスト・AIリスト・架電履歴・予約架電（List item × N）
- **メイン**: サブTABヘッダ、メインコンテンツ

### 9.3 営業ルーム 未調査リスト精査モード

- **左**: 承認ボタン…、未承認リストステータス
- **右**: HP・INFO・検索窓
- **フッタ**: 共通

### 9.4 運営ダッシュボード（バックヤード）

- 共通レイアウト（ヘッダ・カラム・メイン・フッタ）
- 左カラム: 運営メニュー（運営ダッシュボード、課金リミッター/課金plan調整 等）
- メイン: 概要KPI・アラート・今日のアクション

### 9.5 課金リミッター / 課金plan調整

- 運営レイアウトのメインに、課金リミット・プラン変更UIを表示。
